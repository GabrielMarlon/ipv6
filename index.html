Como funciona
01
Sem servidor
Todos os testes rodam diretamente no navegador. Nenhum dado é enviado para servidores próprios. Apenas APIs públicas são consultadas (ipify, ipapi.co, Cloudflare DoH).
02
Testes em paralelo
Os cinco testes de conectividade são executados em paralelo via Promise.all, minimizando o tempo total. O teste de MTU aguarda os anteriores por natureza da medição de throughput.
03
Modelo RF simplificado
O simulador de cobertura usa FSPL (Friis) com atenuação por parede via ray-casting segmento a segmento. Útil para comparação relativa; não substitui medições reais com rádio.
